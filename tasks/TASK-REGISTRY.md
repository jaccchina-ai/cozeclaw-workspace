#### 🎯 任务目标

基于涨停板策略的A股龙头股选股系统，在交易日T日晚间和T+1日早盘进行两次筛选，推荐最有望继续涨停的股票。

#### 🔄 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                    T01: A股龙头选股策略                      │
└─────────────────────────────────────────────────────────────┘

Step 1: 交易日判断
├─ 时间: 任意时间
├─ 数据: 交易日历
└─ 输出: 是否为交易日

         ↓ 如果是交易日

Step 2: T日晚间初步筛选 (T日 20:00)
├─ 输入: 涨停股列表、涨停原因、板块信息、资金流向
├─ 处理: 初步评分
└─ 输出: 
   ├─ 10只观察标的
   ├─ 判断理由
   ├─ 热门板块分析
   └─ 宏观风险分析

         ↓

Step 3: T+1日早盘最终推荐 (T+1日 09:25)
├─ 输入: 竞价数据、大盘走势、板块热度、资金动向
├─ 处理: 结合风险分析进一步筛选
└─ 输出:
   ├─ 3只推荐股票
   ├─ 仓位建议
   ├─ 判断理由
   └─ 风险提示
```

#### ⚙️ 配置参数

| 参数 | 值 | 说明 |
|------|-----|------|
| **交易日历来源** | Tushare | 使用 Tushare API 获取交易日历 |
| **晚间筛选时间** | 20:00 | T日晚上8点执行初步筛选 |
| **观察标的数量** | 10只 | T日晚间筛选出的股票数量 |
| **早盘选择时间** | 09:25 | T+1日早上9:25获取竞价数据 |
| **最终推荐数量** | 3只 | T+1日早盘最终推荐的股票数量 |
| **时区** | Asia/Shanghai | 中国标准时间 |

## 📝 使用指南

### 如何添加新任务

1. 在 `TASK-REGISTRY.json` 的 `tasks` 数组中添加新任务对象
2. 在 `TASK-REGISTRY.md` 中添加对应的 Markdown 文档
3. 在 `tasks/` 目录下创建任务详细文档目录
4. 更新 `statistics` 统计数据

### 如何更新任务版本

1. 修改 `tasks[].version` 字段
2. 在 `tasks[].versionHistory` 中添加新版本记录
3. 更新 `tasks[].updatedAt` 时间戳
4. 更新 `meta.lastUpdated` 时间戳
5. 同步更新 `TASK-REGISTRY.md` 文档

### 如何查询任务

```bash
# 查看所有任务
cat /workspace/projects/workspace/tasks/TASK-REGISTRY.json | grep -A 5 '"id"'

# 查看特定任务
cat /workspace/projects/workspace/tasks/TASK-REGISTRY.json | grep -A 50 '"id": "T01"'
```
